# 렌탈 상품 지속 운영 기준

## 상품 추가 순서

1. 제조사·공식 판매처에서 정확한 모델명, 월 렌탈료, 약정, 관리 방식, 특징을 확인합니다.
2. 제품 단독 이미지를 준비해 관리자 `/admin/rentals`에서 업로드합니다.
3. 상품 ID는 영문 소문자·숫자·하이픈으로 고정하고 이후 변경하지 않습니다.
4. 정보 확인이 끝난 상품만 `verified`, 고객 노출 가능 상태로 변경합니다.
5. 여러 상품은 관리자 CSV 양식으로 일괄 등록합니다.

## 이미지 기준

- 권장 원본: 정사각형 또는 4:3, 600px 이상
- 권장 형식: WebP, PNG, JPG
- 브랜드 로고·가격 문구·워터마크 없는 제품 단독 이미지
- 제품이 카드 이미지 영역의 약 70~78%를 차지하도록 중앙 정렬
- 파일당 6MB 이하

## 정기 점검

```powershell
npm run audit:rental
```

검수 결과는 `docs/rental-catalog-audit.csv`에도 저장됩니다. 중복 ID, 없는 로컬 이미지, 검증 완료 상품의 필수정보 누락이 있으면 명령이 오류로 종료됩니다.

## Supabase 반영

- 전체 로컬 카탈로그 반영: 관리자에서 `기본 상품 DB 동기화`
- 일부 상품만 반영: 관리자에서 CSV 등록·업데이트
- 관리자에서 직접 수정한 `verified` DB 상품은 DB 값을 우선 사용합니다.
- 로컬에 새로 추가한 상품은 DB 동기화 전에도 기존 DB 상품 뒤에 안전하게 함께 노출됩니다.
